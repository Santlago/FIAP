TIPO DE DADOS


TIPO DE DADO NUMERICO: ARMAZENAR NUMEROS INTEIROS OU DECIMAIS, POSITIVOS, NEGATIVOS OU ZERO.
PADRAO (LING. SQL) -> NUMERICO (P, S)
P = PRECISAO = QUANTIDADE TOTAL DE DIGITOS DE UM NUMERO INCLUINDO A PARTE DECIMAL;
S = ESCALA = QUANTIDADE DE DIGITOS DECIMAIS DO NUMERO.

ORACLE -> NUMBER(P, S)
PRECISAO = QUANTIDADE TOTAL SEM FORMATACAO
ESCALA = DECIMAL DO NUMERO

EXEMPLO: NUMERO INTEIRO -> QT_VENDIDA = 1250 (4 DIGITOS DE PRECISAO E ESCALA = 0)
QT_VENDIDA -> NUMERIC(4,0) OU NUMERIC(4) SQL
              NUMBER(4,0) OU NUMBER(4) ORACLE

EXEMPLO NUMERO DECIMAL -> SALARIO MENSAL = R$3457,94 (6 DIGITOS DE PRECISAO E ESCALA = 2)
VL_SALARIO_MENSAL -> NUMERIC(6,2) SQL
                     NUMBER(6,2) ORACLE


TIPO DE DADO PARA ARMAZENAR DATE E HORA
PADRAO (LING. SQL)
    DATE -> ARMAZENAR DATA (DD/MM/YYYY)
    TIME -> ARMAZENAR HORA (HH:MI:SS)

ORACLE -> DATE -> ARMAZENA DATA E HORA (DD/MM/YYYY HH:MI:SS)
OCUPA 7 BYTES

OUTRO TIPO DE DADO QUE TAMBEM ARMAZENA DATA E HORA, COM UMA PRECISAO MAIOR NA PARTE NA PARTE FRACIONARIA DOS MILESIMOS DE SEGUNDO
TIMESTAMP -> DATA E HORA (DD/MM/YYYY HH:MI.SS.sssssssss)
OCUPA 11 BYTES


TIPO DE DADOS TEXTO (STRING/ALFANUMERICO): ARMAZENA CADEIA DE CARACTERES (LETRAS, NUMEROS E CARACTERES ESPECIAIS)

PADRAO (LING. SQL)
    CHAR() -> CADEIA DE CARACTERES DE TAMANHO FIXO
    VARCHAR() -> CADEIA DE CARACTERES DE TAMANHO VARIAVEL
ORACLE  
    CHAR() -> CADEIA DE CARACTERES DE TAMANHO FIXO
        CAPACIDADE: 2000 BYTES
    SUGESTAO DE USO: PARA TUDO ONDE O TAMANHO E O CONTEUDO FOR CONHECIDO
        SIGLA DOS ESTADOS (SP, MG, RJ...) -> CHAR(2)
        GENERO (H, M, NB) -> CHAR(2)
        SEXO (M,F) CHAR(1)
        PLACA VEICULO (FJI9H34) -> CHAR(7)
    VARCHAR2() -> CADEIA DE CARACTERES DE TAMANHO VARIAVEL
        CAPACIDADE: 4000 BYTES, PODENDO ESTENDER ATE 32767 BYTES (v12c)
    SUGESTAO DE USO: TUDO
